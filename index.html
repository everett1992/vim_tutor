Vim can be a confusing editor to learn. If you've ever opened Vim and begun typing, you were probably confused when you didn't see any text appear. This is because Vim relies on modes. Once you learn to speak to Vim it will become the only editor you want to use.

Vim The modal editor

Normal Mode:
Keys are mapped to commands. This is the default mode. Pressing esc from any other mode will return you to this mode. This is the mode you should spend the majority of your time in.
Insert Mode:
Typing a key will insert that character into the document. Press i to enter insert mode.
Visual Mode:

Highlight groups of text and perform commands on the selection. There are three sub-modes: Visual, Visual-Line, and Visual-Block which can be entered using v, V , and ctrl+v respectively.

The most basic way of using Vim is to immediately enter insert Mode upon opening a file, navigating using the arrow keys, and never returning to Normal Mode until exiting the file. Using Vim this way makes using Vim like using a worse Notepad.

Talking with your editor Vim's command syntax.

Vim's commands can be chained using a simple grammar. Most Vim commands can be classified as motions, actions, or modifiers.
Motions

Vim uses h, j, k, and l to move left, down, up, or right instead of the arrow keys, this keeps your hands on the home row. ^, and $ will move the cursor to the end of the line.

There are motions for more advanced movements like next word, beginning of word and end of word, (w, b, and e) and the area between quotes, brackets, and tags (", [, and t). There are some motions that take arguments like f and t which move the cursor to the entered key, or before the entered key. fx will move your cursor to the next 'x' that appears in the document, while tx would place the cursor to the character before the next 'x'. You can search for text using /, and cycle through the results with n and N.

Actions

Action commands modify text. As mentioned earlier i will place you in insert mode at the current cursor position, There are many ways to enter insert mode:

  • I Inserts text at the beginning of a line,
  • A Inserts text at the end of a line.
  • o Inserts a new line below the current line (O above).
  • C Deletes the text from the cursor to then end of the line and enters insert mode.
  • x Deletes the character under the cursor.
  • u Undoes the previous command.
  • ctrl+r Redo.

Many of Vim's action commands act on motion commands, e.g. the d command deletes motions.

  • dw Will delete from the cursor to the end of the current word.
  • d$ Will delete from the cursor to the end of the current line (D is shorthand for this).

Other action commands that accept motions include change c, and yank y.

  • cw Will delete from the cursor to the end of the current word then enter insert mode.
  • y$ Will yank (copy) the text from the cursor to the end of the line.
As you learn more actions and motions you can use them in combination. If you know what an action does, and what a movement does, you'll know what the application of the action on the movement will do.

Modifiers

Modifiers, such as 'a' and 'inner' (a and i, respectively) enhance the meaning of a motion. Unlike dw that deletes from the cursor position to the beginning of the next word, diw will delete the entire word, even the part before the cursor. There are many motions that can only be invoked with an action and a modifier like ), ], ", t.

  • i) The text between matching parentheses.
  • a) The text between and the matching parentheses.
  • it The inner text of the current HTML style tag.
  • at The whole of the current HTML style tag.
  • iw The text from the beginning of the current word until the end.
  • aw The text from the beginning of the current word, until the start of the next word.

Motions can also be modified by numbers so d4w will delete four words.

Working Semantically Working at the speed of thought.

You may have noticed that there are different ways to accomplish the same task in Vim. cw does the same thing as dw followed by i. You could even mash x to delete each character under the cursor until the word is deleted, then press i. cw is one discrete command, one u will undo the whole action while other commands would require multiple undos. Vim remembers your last command, and you can repeat it with .. See how below we use ciw to make a repeatable action, which we then repeat with ..

If you want to change a functions arguments in less sophisticated editors you would use the mouse or arrow keys to move to the first parentheses, then press delete until you have reached the next parentheses. In vim you can do this using f(ci). Then if you have to replace the same arguments again you can just press ..
